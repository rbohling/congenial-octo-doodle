GitHub Pages Prep Runbook

1. Repository Setup

Description: Prepare a new GitHub repository for your site.

  1. Create a new repository in your GitHub account (e.g., username/username.github.io or a project repository).
  2. Clone the repository locally and initialize the working directory.
  3. Add an initial README.md and commit the changes.

2. Local Workspace

Description: Set up the local environment for working on the site.

  1. Ensure Git is installed and configured on your system.
  2. Install a code editor (e.g., VS Code) if not already available.
  3. Create the directory structure for your site (e.g., folders for assets, css, images).

3. Create Your First Page

Description: Build a basic landing page to kick-start your site.

  1. Create an index.html file in the repository.
  2. Add a basic HTML structure with a title, header, navigation, content section, and footer.
  3. Link to a CSS file (e.g., styles.css) for styling placeholders.
  4. Commit and push the page to GitHub.

4. Configure GitHub Pages

Description: Enable GitHub Pages and select the branch and folder to publish.

  1. Go to your GitHub repository settings.
  2. Under “Pages,” select the branch (e.g., main) and folder (root or docs) to publish.
  3. Save the settings and note the URL provided by GitHub Pages.

5. Verify Deployment

Description: Check that your site is live and rendering correctly.

  1. Open the GitHub Pages URL in a web browser.
  2. Verify that your index.html page loads and displays as expected.
  3. Troubleshoot any issues (e.g., broken links, missing styles).

6. Maintenance & Updates

Description: Plan for ongoing updates and improvements.

  1. Add additional pages (e.g., about.html, contact.html) as needed.
  2. Set up a branch workflow for developing and testing changes before merging to main.
  3. Periodically review the site for outdated information and broken links.
