Standard Operating Procedures

Add a New VLAN

  1. Open a change ticket.
  2. Define VLAN ID, subnet, gateway, and DHCP scope.
  3. Create the VLAN on the core switch and configure trunk ports.
  4. Establish firewall rules to allow or restrict traffic as needed.
  5. Create a DHCP scope and verify DNS registration.
  6. Test connectivity from a host on the new VLAN.
  7. Update documentation including this runbook.

Add a Firewall Rule

  1. Confirm the business requirement and least‑privilege needed.
  2. Identify source IP/VLAN, destination IP/VLAN, ports, and protocols.
  3. Implement the rule on the firewall with logging enabled.
  4. Test connectivity and verify logs.
  5. Record the rule details in documentation, including ticket number and expiration date if applicable.

Rotate VPN Credentials

  1. Notify all users of the upcoming change and provide instructions.
  2. Generate new credentials and update any multi‑factor authentication policies.
  3. Revoke old credentials after verifying the new ones work.
  4. Confirm users can connect successfully and document the process.

Troubleshooting Guides

Internet Outage

  1. Determine the scope: is it affecting a single user or the entire site?
  2. Check the edge firewall’s WAN link status and IP assignment.
  3. From the firewall, ping the ISP gateway, then a public IP (such as 1.1.1.1), and perform a DNS resolution test.
  4. Review firewall logs for signs of WAN link or authentication failures.
  5. If the issue persists, open a ticket with the ISP and note the ticket number.
  6. Communicate status to stakeholders using the appropriate template.

DNS Resolution Fails

  1. Check the health of the DNS server (SRV‑DNS‑01).
  2. Ensure clients have the correct DNS server configured via DHCP.
  3. Use nslookup or dig to test resolution directly against the DNS server.
  4. Verify firewall rules allow DNS traffic between VLANs.
  5. Restart the DNS service if necessary and examine event logs for errors.

Guest Wi‑Fi Can’t Access Internet

  1. Verify the guest SSID is active and devices can connect.
  2. Check that the DHCP scope for VLAN 40 is issuing addresses.
  3. Ensure firewall rules allow outbound Internet traffic from the guest VLAN.
  4. Inspect captive portal settings if in use.
  5. Test the guest network with a known good device and collect evidence.
