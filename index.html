Design Notes

Layout rationale: the page uses a named CSS Grid to separate global scaffolding (sidebar, header, main, footer). The sidebar holds navigation and identity; this keeps primary actions reachable and allows the main column to scale horizontally. Internally components (navigation and card lists) use Flexbox to adapt row/column flows across breakpoints.

Scalability: all colors, spacing and sizes are expressed as CSS variables in :root. To re-theme or scale spacing, update only root variables and all components respond instantly. Breakpoints adjust grid areas instead of reworking components, keeping component CSS portable.
