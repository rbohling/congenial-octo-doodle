6) Standard Operating Procedures

6.1 Add a New VLAN (Change-Control Required)

  1. Open change ticket: [Ticket #]
  2. Define VLAN ID, subnet, gateway, DHCP scope requirements.
  3. Create VLAN on core switch and required trunks.
  4. Create firewall rules (allow/deny between VLANs).
  5. Create DHCP scope (if needed) and confirm DNS registration.
  6. Test: ping gateway, DNS resolve, internet access (if allowed).
  7. Update documentation: VLAN table + diagram.

6.2 Add a Firewall Rule

  1. Confirm business need and least-privilege requirement.
  2. Identify source VLAN/IP, destination VLAN/IP, ports/protocols.
  3. Implement rule with logging enabled.
  4. Test connectivity from a test host.
  5. Document: rule name, reason, ticket #, expiry (if temporary).

6.3 Rotate VPN Credentials (Quarterly)

  1. Notify users: [Email Template Placeholder]
  2. Generate new credentials / update MFA policy.
  3. Revoke old credentials.
  4. Validate remote login.
  5. Close ticket with evidence (screenshots/log export).

7) Troubleshooting Runbooks

7.1 Internet Outage (All Users)

  1. Confirm scope: one user vs whole site.
  2. Check edge firewall WAN status (link up? IP assigned?).
  3. Ping tests (from firewall if possible):
    • Ping ISP gateway: [ISP GW IP]
    • Ping public IP: 1.1.1.1
    • DNS test: resolve example.com
  4. Check logs for WAN down or authentication errors.
  5. If WAN down: open ISP ticket and record ticket #.
  6. Update status page / send internal comms: [Template Placeholder]

7.2 Users Can Ping But DNS Fails

  1. Check DNS server health: SRV-DNS-01 availability.
  2. From a client: confirm DNS server set correctly (DHCP options).
  3. Test resolution using nslookup/dig against DNS server.
  4. Check firewall rules for DNS (UDP/TCP 53) between VLANs.
  5. If DNS service down: restart service, review event logs.
  6. Document incident and root cause.
